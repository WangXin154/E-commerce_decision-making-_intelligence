# 模块 02 —— 用户分群与价值分析（RFM 模型）

## 模块概述

本模块基于 **RFM** 方法构建面向业务的用户分群体系，即通过 **Recency（最近购买）**、**Frequency（购买频率）** 和 **Monetary（消费金额）** 三个维度，将原始用户行为聚合结果转化为可执行的客户分层，用于支持留存策略、激活策略、价值优先级判断以及后续个性化应用。

与教材式的标准 RFM 不同，本模块根据数据本身的行为分布特点，对评分逻辑进行了专门修正，因为该数据集中购买频次分布极度偏斜，大多数用户都只购买过一次。

---

## 业务问题

本模块主要回答以下几个核心问题：

1. 哪些客户群应该被差异化留存或激活？
2. 客户价值在整个用户群中是如何分布和集中的？
3. 基于 RFM 的行为分层，能否成为一个在统计上可靠、在业务上可解释的策略基础？

从业务角度看，本模块的作用是把用户购买行为转化为可执行的运营分群，为 CRM、用户激活和价值优先级管理提供支持。

---

## 数据来源

本模块基于项目数据仓库中的用户行为聚合视图展开分析，核心对象是用户级购买历史。

主要变量包括：

- `unique_user_id`
- `recency`
- `frequency`
- `monetary`
- `state`
- 用户活跃度和消费汇总指标

分析时仅保留至少有一次购买记录的用户。

## 方法论

### 1. 数据准备
本模块从用户级行为聚合数据开始，先筛选有效购买用户。

初始数据概况如下：
- `order_count > 0` 的用户总数：**96,219**
- `monetary` 缺失值：**680**
- 删除缺失 RFM 记录后的最终用户数：**95,539**

随后统一字段命名为：
- `recency`
- `frequency`
- `monetary`

### 2. 探索性分析与数据诊断
在正式分群前，本模块先检查：
- 缺失值
- 异常值
- 偏态分布
- 非正态性
- 累积分布不均衡现象

关键观察包括：
- `frequency` 极度集中在 1 次购买
- `monetary` 呈明显右偏
- `recency` 具有较明显的业务区分度
- 异常值被保留，因为它们可能代表重要的高价值或特殊用户群体

基于 IQR 的异常值检测结果为：
- recency 异常值：**3**
- frequency 异常值：**2,878（3.01%）**
- monetary 异常值：**7,612（7.97%）**

### 3. 改进版 RFM 评分逻辑
由于该数据集中绝大多数用户的 `frequency = 1`，如果直接对频次做常规分位数打分，会导致分层效果失真。

因此本模块采用了改进版评分方式：

- **Recency**：按分位数从 5 到 1 打分，越近期购买得分越高
- **Frequency**：
  - 所有 `frequency = 1` 的用户统一赋值 **F_score = 1**
  - 仅对 `frequency >= 2` 的复购用户进一步划分 **2–5 分**
- **Monetary**：基于排序后的分位数从 1 到 5 打分

之后构造两个总分：
- `RFM_score_simple`
- `RFM_score_weighted`

加权分数定义为：

- **0.4 × R + 0.1 × F + 0.5 × M**

这个设计的目的，是在复购极度稀少的前提下，弱化频次、强化金额，使分数更贴近真实业务价值。

### 4. 聚类分析
在构建好 RFM 特征后，本模块对 RFM 特征进行标准化，并使用 K-Means 进行聚类。

聚类效果评估使用：
- Elbow Method
- Silhouette Score
- Davies–Bouldin Index
- Calinski–Harabasz Score

最终选择的聚类数为：**K = 3**

聚类质量指标为：
- **Silhouette score = 0.4557**
- **Davies–Bouldin = 0.7619**
- **Calinski–Harabasz = 48636.7004**

### 5. 统计验证
为了保证分群结果不仅“看起来像”，本模块进一步使用统计方法验证分群差异：

- Kruskal–Wallis 检验
- Mann–Whitney U 两两比较（Bonferroni 校正）
- RFM 分数与 monetary 的 Spearman 相关
- geography 与 cluster 的卡方检验

这使得分群结果具备统计支撑，而不只是可视化上的区分。

### 6. 业务翻译
聚类完成后，本模块将每个 cluster 翻译为更易理解的业务标签：

- **Cluster 0** → Lapsed Single-Purchase Users
- **Cluster 1** → Recent Single-Purchase Users
- **Cluster 2** → High-Value Repeat Customers

在此基础上，本模块继续扩展到：
- 价值集中度分析
- 流失风险启发式判断
- 策略矩阵
- A/B 测试设计
- 高价值用户的简化 LTV 估算

## 核心发现

### 1. 用户群体以单次购买用户为主
数据表明，购买频次分布极度集中，大多数用户只购买过一次，这也是本模块必须重构 Frequency 打分逻辑的根本原因。

这本身就是一个重要业务发现：平台的复购基础较弱，因此“首购转复购”是非常关键的增长问题。

### 2. 三分类方案形成了清晰且可解释的行为分层
最终分群结果如下：

- **Cluster 0**：39,363 人（**41.20%**）
- **Cluster 1**：53,291 人（**55.78%**）
- **Cluster 2**：2,885 人（**3.02%**）

各群体平均 RFM 特征如下：

| Cluster | Recency | Frequency | Monetary |
|---|---:|---:|---:|
| 0 | 391.80 | 1.00 | 162.27 |
| 1 | 131.59 | 1.00 | 159.81 |
| 2 | 224.04 | 2.11 | 325.69 |

这形成了三个很清晰的业务角色：
- 较久未活跃的单次购买用户
- 最近刚买过一次的单次购买用户
- 数量少但更高价值的复购用户

### 3. 分群差异具有显著统计意义
Kruskal–Wallis 检验结果表明，三个 cluster 在核心 RFM 维度上都存在显著差异：

- recency：**H = 67989.9210**, **p ≈ 0**
- frequency：**H = 95292.1431**, **p ≈ 0**
- monetary：**H = 2366.6007**, **p ≈ 0**

这说明聚类结果不是随机聚合出来的，而是具有明确统计差异的行为群体。

进一步的 Mann–Whitney 两两比较也表明：
- recency 在所有 cluster 之间都显著不同
- frequency 的显著差异主要来自 Cluster 2 与其他群体
- monetary 在 cluster 间同样存在显著差异

### 4. 加权 RFM 分数与真实用户价值更一致
RFM 分数与 monetary 的 Spearman 相关结果为：

- simple RFM vs monetary：**rho = 0.6978**
- weighted RFM vs monetary：**rho = 0.7826**

这说明加权版 RFM 比传统简单总分更能反映真实消费价值，因此更适合作为业务评分层使用。

### 5. 分群与地域之间存在弱但显著的关系
`state × cluster` 的卡方检验结果为：

- **chi2 = 364.8234**
- **dof = 52**
- **p = 1.5096e-48**

这说明地理因素不是决定性变量，但不同地区的用户结构与 cluster 分布之间确实存在统计上的联系。

### 6. 用户价值分布不均衡，但并非只集中在最小群体中
各群体的用户占比和 GMV 占比如下：

| Cluster | 用户占比 | GMV 占比 |
|---|---:|---:|
| 0 | 41.20% | 40.32% |
| 1 | 55.78% | 53.75% |
| 2 | 3.02% | 5.93% |

Cluster 2 在单用户价值上明显更高，但由于用户规模很小，Cluster 1 仍然贡献了最大的总 GMV。

这意味着业务上不能只关注 VIP 留存，也必须重视大规模的首购用户激活。

### 7. Cluster 0 具有最高的“流失式不活跃风险”
以 recency 的 75% 分位数作为高流失风险阈值时：

- churn threshold：**351 天**
- Cluster 0 流失率：**59.24%**
- Cluster 1 流失率：**0.00%**
- Cluster 2 流失率：**21.21%**

notebook 还估算出：如果成功挽回 Cluster 0 中 10% 的流失用户，预计可带来约：

- **376,440.11** 的 GMV 增量

这使 Cluster 0 成为非常重要的召回目标群体。

### 8. Cluster 1 是最适合做二次购买激活实验的群体
Cluster 1 代表最近刚购买过一次的用户，因此最适合做首购转复购的实验。

notebook 中的 A/B 测试设计假设为：
- 基线转化率：**5%**
- 目标转化率：**7%**
- 最小可检测提升：**2 个百分点**
- 每组所需样本量：**30,913**
- 四组实验总样本量：**123,652**

而当前 Cluster 1 可用用户数为：
- **53,291**

这说明严格按 4 组设计做实验时样本量不足，但这个实验框架本身对未来 CRM 运营依然非常有价值。

### 9. Cluster 2 支持明确的 VIP 留存投资逻辑
针对 Cluster 2（高价值复购用户），notebook 估算得到：

- 平均客单价：**163.49**
- 平均购买频率：**2.11**
- 平均生命周期：**85 天**
- 估算年购买频次：**9.10**
- 调整后 LTV：**2902.18**
- 全体客户平均 LTV：**165.83**
- Cluster 2 的 LTV 溢价：**17.5 倍**

这说明为该群体提供更高触达强度、更高服务等级的 VIP 运营是有充分业务依据的。

## 分群业务解读

### Cluster 0 —— Lapsed Single-Purchase Users
**画像**
- 最近购买时间较久
- 单次购买
- 消费额中等
- 不活跃风险最高

**业务意义**
这类用户曾经购买过，但目前已明显沉寂，是主要的召回池。

**建议动作**
- win-back 召回活动
- 提醒类触达
- 定向优惠
- 低成本自动化挽回策略

---

### Cluster 1 —— Recent Single-Purchase Users
**画像**
- recency 最低，即最近刚买过
- 单次购买
- 消费水平与 Cluster 0 接近
- 用户规模最大

**业务意义**
这是最值得做“首购转复购”的增长型人群。

**建议动作**
- 二次购买激活活动
- 新客 onboarding
- 下单后优惠券 / 邮件流
- 可控实验设计

---

### Cluster 2 —— High-Value Repeat Customers
**画像**
- 购买频率最高
- 消费金额最高
- 人数很少
- 单用户经济价值最强

**业务意义**
这是核心高价值客户群，应重点保护并进一步提升客单与忠诚度。

**建议动作**
- VIP 留存
- 高级服务
- 优先客服
- 忠诚度权益
- 更高价值的个性化推荐

## 业务建议

基于本模块结果，建议采取以下策略：

1. **高效率召回 Cluster 0**  
   优先识别最有可能被召回的不活跃用户，用低成本方式进行挽回。

2. **把 Cluster 1 作为复购增长主战场**  
   这是最适合做实验和提升二次购买率的人群。

3. **重点保护并扩展 Cluster 2**  
   用更高质量的服务和权益机制维持高价值关系，并推动进一步消费。

4. **将加权 RFM 分数作为业务评分层复用**  
   由于加权分数更贴近实际价值，它可以被后续推荐系统、流失预测或 CRM 排序逻辑复用。

5. **让分群结果服务于下游决策系统**  
   本模块结果应进一步连接到流失预测、CLV、Dashboard 和推荐系统的个性化路由中。
